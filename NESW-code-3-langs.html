3 Languages in 30 Minutes

Say hello to HTML, CSS, and JavaScript

Yes, it's true. Just to build a single little web page, you need to learn at least three languages. HTML (Hypertext Markup Language) sets out the basic structure of the document—its content, its substance, its bones. CSS (Cascading Style Sheets) dresses up the page, defining how things look. JavaScript brings action and interaction, setting the machinery in motion. Think of the three languages this way: HTML = nouns, CSS = adjectives and adverbs, JS = verbs.

Can you really learn three languages in half an hour? Maybe not, but ... not become a fluent native speaker, but you can learn enough to survive as a tourist. We have amazing tools. Fortunately, computer languages are a good deal easier to learn than human ones. Also, in writing the HTML and CSS components of a web page you seldom need to start from scratch; there's readily available boilerplate you can adapt or customize. Hence in our half-hour tutorial we'll pause only briefly on HTML and CSS, then move on to JavaScript.

Building stuff with HTML

Here is a snippet extracted from the HTML file for the web page you are reading right now:

<div id="content"> 
    <h2>3 Languages in 30 Minutes</h2>
    <h3>Say hello to HTML, CSS, and JavaScript</h3>
    <img src="3langs-logos.png" width="360px" height="396">
    <p>Yes, it's true. Just to make a silly web page, . . . </p>
    <h4>Building stuff with HTML</h4>
    <p>Here is a snippet of the HTML file for the web page you are 
    reading right now:</p>
    . . .
</div> <!--"content"-->

HTML is all elbows, bristling with angle brackets: '<' and '>'. The brackets enclose tags, such as <p>, and matched pairs of tags, such as <p> ... </p>, mark the structure and

  • <p> ... </p>: paragraph
  • <div> ... </div>: division, an arbitrary block of text or other content that you want to treat as a unit
  • The tags need to be properly nested, like parentheses.
